# Checkbox 紀律

checkbox 是進度紀錄，不是承諾或樂觀預估。

## 何時可以勾 `[x]`

至少同時滿足：
- 該步驟的實際工作已完成
- 已有對應證據，例如檔案、命令輸出、結果表或 report 更新
- 後續不需要再回頭補同一個核心交付物

## 何時不能先勾

以下情況不要先勾：
- 只差最後整理，但還沒整理完
- 實驗還在跑，或結果還沒檢查
- report 還沒更新
- 目前只有主觀判斷，沒有可追溯證據

## 缺步驟時怎麼做

若執行時發現原 checklist 不完整：
- 在原條目下補新的 checkbox 或子 bullet
- 讓新增步驟保持可見
- 不要把缺的工作默默吸收到既有 checkbox 裡

## 部分完成

若只完成一部分：
- 保持 `[ ]`
- 必要時在旁邊補一句目前卡在哪裡
- 或拆成更小的 checkbox，再如實更新
