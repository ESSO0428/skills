# 完成規則

「整個計畫完成」與「某一步有進展」不是同一件事。

## 何時可視為整個計畫完成

至少同時滿足：
- `ROADMAP.md` 這次主條目下的主要 checkbox 已完成
- 正式結果已寫入 `docs/reports/*.md`
- 重要表格已有資料來源
- report 已補齊以下段落：
  - `結果`
  - `結論`
  - `討論`
  - `最終結論`

## 尚未完成的常見情況

以下都不應宣告整個計畫已完成：
- 實驗還沒全跑完
- 只留下設定、命令或表格，還沒有論述
- 表格沒有資料來源
- `ROADMAP.md` checkbox 已勾，但 report 尚未更新

## 收尾時的最低要求

在說「這個計畫完成了」之前，至少要能回答：
- 做了哪些實驗
- 結果是什麼
- 為什麼這樣解讀
- 最後的方向判斷是什麼

若這四個問題還答不完整，就代表應先繼續補 report，而不是先把計畫視為完成。
